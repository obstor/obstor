# S3 Protocol

S3 is the primary protocol for Obstor. It provides full compatibility with the Amazon S3 API, allowing any S3-compatible client or SDK to interact with your object storage.

## Features

- Full S3 API compatibility
- Multi-part upload support
- Server-side encryption (SSE-S3, SSE-C)
- Bucket versioning, lifecycle, and replication
- IAM policies and access control
- Presigned URLs

## Quick Start

The S3 protocol is enabled by default on port 9000:

```sh
export OBSTOR_ROOT_USER=admin
export OBSTOR_ROOT_PASSWORD=password
obstor server /data --s3-address :9000
```

## Client Compatibility

Any S3-compatible client works with Obstor:

| Client | Platform |
|--------|----------|
| AWS CLI | Cross-platform |
| rclone | Cross-platform |
| AWS SDK (all languages) | Cross-platform |
| s3cmd | Cross-platform |
| Cyberduck | macOS, Windows |

## Access Control

S3 access is controlled through IAM policies. Here's an example policy that grants full S3 access and limited admin access to a specific bucket:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "admin:DataUsageInfo",
        "admin:GetBucketQuota",
        "admin:GetBucketTarget",
        "admin:Heal",
        "admin:SetBucketTarget",
        "admin:TopLocksInfo"
      ],
      "Resource": [
        "arn:aws:s3:::examplebucket*"
      ]
    },
    {
      "Effect": "Allow",
      "Action": [
        "s3:*"
      ],
      "Resource": [
        "arn:aws:s3:::examplebucket*"
      ]
    }
  ]
}
```

### Creating an S3 User

Create the user, define the policy, and attach it. This adds an IAM user, registers a named policy from your policy document, and binds that policy to the user.

The user can then access the bucket via any S3 client:

```sh
aws s3 ls s3://examplebucket --endpoint-url http://your-server:9000
```
